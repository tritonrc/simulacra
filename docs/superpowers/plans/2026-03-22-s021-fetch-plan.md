# S021 Fetch API & HTTP Control Plane — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the toy `fetch()` in simulacra-quickjs with a WHATWG-aligned Fetch API backed by a shared HTTP control plane.

**Architecture:** Two new crates: `simulacra-http` (HTTP client trait + ureq implementation, no agent/JS concepts) and `simulacra-fetch` (WHATWG JS classes: Headers, Request, Response, Blob, AbortController/AbortSignal as rquickjs-native Rust structs). `simulacra-sandbox` implements `FetchProxy` to bridge the Golden Rule. The old inline fetch code in `simulacra-quickjs` is deleted.

**Tech Stack:** Rust, rquickjs (class macros), ureq, serde/serde_json, thiserror, tracing

**Spec:** `docs/superpowers/specs/2026-03-22-s021-fetch-design.md`

**Key references:**
- `crates/simulacra-sandbox/src/http.rs` — current `do_http_request()` and `fetch_http_inner()` to refactor
- `crates/simulacra-quickjs/src/lib.rs:195-223` — current `FetchProxy` trait and `FetchResponse` to move
- `crates/simulacra-quickjs/src/lib.rs:968-1076` — current inline fetch JS to delete
- `rules/R004-test-against-fakes.md` — test patterns (use fakes, no real HTTP in unit tests)
- LLRT (github.com/awslabs/llrt) — reference patterns for Headers guard, BodyVariant, ModuleBuilder

**Deferred:** `crates/simulacra-http/src/policy.rs` (request policy hooks — rate limits, circuit breakers) is called out in the spec's crate structure as "future" and will NOT be created in this implementation. The `HttpClient` trait is designed so a policy layer can wrap `UreqHttpClient` later without changing callers.

- [ ] **Step 1: Implement `register_fetch_global(ctx, proxy)`**

Registers `fetch` on JS global. The function:
1. Parses input (string URL or Request object)
2. Merges init overrides if input is Request
3. Checks signal.aborted — rejects immediately if true
4. Extracts timeout_ms from signal
5. Calls proxy.fetch(url, method, headers, body, timeout_ms)
6. Constructs Response from FetchResponse
7. Returns Promise.resolve(response) or Promise.reject(error)
